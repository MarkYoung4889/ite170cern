Here are a few questions about the underlying connectivity of a hypertext web.

Are links two- or multi-ended?

The term "link" normally indicates with two ends. Variations of this are links with multiple sources and/or multiple destinations, and constructs which relate more than two anchors. The latter map onto logic description systems, predicate calculus, etc. See the "Aquanet" system from Xerox PARC - paper at HT91). This is a natural step from hypertext whose links are typed with semantic content.

Should the links be monodirectional or bidirectional?

If they are bidirectional, a link always exists in the reverse direction. A disadvantage is constraint on authors, but an advantage is automatic inverse relationships.

KMS and hypercard have one-way links; Enquire has two-way links.

See also: Private overlaid web, Generic Links.

See Building Back-links.

Should anchors have more than one link?

There is a design issue in whether one anchor may lead to many links... JFG disagrees.

Should links be typed?

A typed link carries semantic information. See list of some types.

Should links contain ancillary information?

Discussion about whether links should contain metadata such as authorship or comments.

Should a link contain preview information?

Preview information may be cached or included by the author, but is not required.
