Hypertext Terms

This is a glossary of terms used within the WorldWideWeb project.

Anchor
An area within the content of a node which is the source or destination of a link.
Annotation
The linking of a new commentary node to an existing node within the web. Annotation and multiuser collaboration
Browser
A program which allows a person to read hypertext, view nodes, and navigate between them.
Button
An anchor which is the source of a link.
Card
An alternative term for a node.
Client
A program which requests services of another program. Normally, the browser is a client.
Database
A collection of nodes managed by a server with internal links.
Daemon
A program which runs independently of the browser or server, performing tasks such as building indexes and back-links.
Document
A term for a node.
Domain
A unit of protection, often related to a database.
External
A link to a node in another database.
Host
A computer on a network, not to be confused with a node in a web.
Hypermedia
Multimedia Hypertext. Hypermedia overview
Hypertext
Text which is not constrained to be linear. Hypermedia overview
Index
A structure which points to data. Index design considerations
Internal
A link to a node within the same database.
Link
A relationship between anchors within a database. See Internal and External.
Navigation
The movement between nodes in the web using links and a browser. Navigation techniques and tools
Node
A unit of information, often referred to as a document.
Path
An ordered set of nodes within a web.
Reader
A person using a browser.
Server
A program which provides services to a client, delivering hypertext to a browser. See also: daemon.
Tracing
Automatic discovery of nodes through navigation.
Topology
The structure of connections between nodes. Topology design issues
Versioning
Managing multiple versions of information. Versioning design issues
Web
A set of nodes connected by links. See also Topology.
